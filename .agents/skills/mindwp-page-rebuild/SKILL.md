# MindWP Page Rebuild

Use this skill for page rebuilds and page-level planning. Do not start in JSX/CSS until the page job and section jobs are clear.

## Read First

- `AGENTS.md`
- `docs/FOUNDATION.md` for identity and buyer truth
- `docs/STRATEGY.md` for offer ownership and system boundaries
- `docs/PAGES.md` for page role, IA, CTA posture, and page planning
- `docs/WRITING.md` when public copy is being written or audited
- `docs/ARCHITECTURE.md` when source structure or routing is unclear

Do not read `_dev-reference/` unless the user explicitly asks for it.

## Page Lock

Before section planning, set the page frame:

- page identity motif
- flagship artifact
- value/density rhythm
- proof location
- final CTA posture

Then answer:

- What page type is this?
- What business moment does it own?
- What is slipping there?
- What does MindWP change?
- What proof or artifact makes the claim believable?
- What is the flagship section?
- What sections are needed, in what order?
- What is the primary CTA and where does it appear?
- What must the page not imply?

If those answers are weak, do not build.

Re-derive a page job when a sibling page has changed. Do not assume the old role still fits.

## Build Sequence

1. Lock the page spine.
2. Lock visual form: silhouette, dominance rank, artifact plan, mobile form, and fail conditions.
3. Lock the styling approach: global foundation changes, page CSS files, artifact CSS, and any justified isolated modules.
4. Build in small chunks.
5. Audit the full page rhythm after each meaningful chunk.
6. Rebuild weak forms instead of polishing them.
7. Keep routes thin and sections owned.

Home is the default taste gate. A narrower page can be rebuilt first only when the user explicitly scopes it.

Do not reopen locked page decisions without new evidence from copy, strategy, implementation, or screenshots.

## Page-Level Checks

- Offer appears early enough.
- Proof appears before strong close.
- Adjacent sections do not repeat the same structure.
- Page styling feels like one coherent system, not unrelated section fragments.
- Service pages keep their system boundary.
- Industry pages are specific to the vertical.
- Implementation pages resolve upward to Smart Website Systems.
- Strong close uses a calm diagnostic CTA.
